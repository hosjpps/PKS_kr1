Практическое задание "Управление библиотекой книг"
Цель: Создать приложение для управления небольшой библиотекой с
использованием Entity Framework Core и WPF.
ЗАДАНИЕ:
Создание базы данных:
1. Создайте следующие сущности:
 Book (Id, Title, Author, PublishYear, ISBN, Genre, QuantityInStock);
 Author (Id, FirstName, LastName, BirthDate, Country);
 Genre (Id, Name, Description).
2. Настройте связи между сущностями:
 Книга может иметь одного автора (связь один-ко-многим);
 Книга может относиться к одному жанру (связь один-ко-многим).
3. Создайте DbContext и настройте конфигурацию сущностей через Fluent
API:
 Определите первичные ключи;
 Настройте обязательные поля; 
 Установите ограничения на длину строковых полей;
 Настройте каскадное удаление.
Создание WPF интерфейса:
1. Создайте главное окно со следующими элементами:
 DataGrid для отображения списка книг;
 Панель фильтрации по жанру и автору;
 Кнопки для добавления, редактирования и удаления книг. 
2. Реализуйте отдельные окна:
 Окно добавления/редактирования книги;
 Окно управления авторами;
 Окно управления жанрами.
3. Добавьте следующий функционал:
 CRUD операции для всех сущностей;
 Поиск книг по названию;
 Фильтрация книг по автору и жанру;
 Отображение количества книг в наличии. 

Практическое задание: Первоначальная настройка git-репозитория

ЦЕЛЬ:
	•	Создание и первичная настройка удалённого и локального репозиториев;
	•	Освоение базовых команд при работе с системой git.

ТЕОРЕТИЧЕСКИЙ МАТЕРИАЛ:
	•	Создать публичный репозиторий в GitHub;
	•	Создать новую ветку test_branch и перейти в неё;
	•	Создать в репозитории папку KR1 и в ней воспроизвести простой по структуре C++ - проект минимум из 3 файлов, например:
	•	hello.h - файл с объявлением функции void printHelloWorld(std::ostream&), выводящей в указанный поток вывода строку “Hello, World!\n”;
	•	hello.cpp - файл с определением вышеуказанной функции;
	•	main.cpp - файл, содержащий главную функцию main(), которая вызывает printHelloWorld(std::cout).
	•	Собрать проект (например, при помощи gcc или Makefile) и запустить исполняемый файл, убедиться, что всё работает корректно;
	•	Добавить все новые файлы в индекс (staging area) коммита;
	•	Выполнить коммит и слить данную ветку с основной;
	•	Произвести отправку изменений в удалённый репозиторий.

Выполнение шагов данного задания следует сопровождать скриншотами гитхаба/командной строки/терминала/IDE, где видны сами шаги и результат их выполнения.
